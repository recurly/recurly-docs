# Overview

Accept payments outside the App Store by integrating Recurly directly into your web or mobile-web environment.

### Prerequisites & limitations

* You must comply with Apple’s guidelines on alternative payment options for digital goods.
* These integrations run in a web context—they don’t replace in-app purchases within native iOS apps.

# Definition

Recurly provides two turnkey solutions to handle payments for digital subscriptions and goods outside of Apple’s native in-app purchase system.

# Key benefits

* **Policy-compliant checkout**: Meets Apple requirements for alternative payment methods without sacrificing user experience.
* **Flexible integration**: Choose a fully hosted flow with Recurly Checkout or build a bespoke form with Recurly.js.
* **Optimized for conversion**: Mobile-friendly designs and secure tokenization help maximize successful transactions.

# Key details

* **[Recurly.js](https://docs.recurly.com/docs/recurlyjs#/)**: A customizable JavaScript library that lets you build your own secure, hosted payment form.
* **[Recurly Checkout](https://docs.recurly.com/docs/checkout#/)**: A prebuilt, mobile-optimized hosted checkout page that launches quickly with minimal setup.

Both options deliver PCI-compliant, flexible payment experiences that live entirely on the web.
