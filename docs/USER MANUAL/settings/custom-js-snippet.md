Admin users may override certain behaviors of the Redfast Javascript SDK. Upon deploying Redfast for the first time, it is important to review configure the following to ensure that users are recognized using your unique identifiers. These identifiers will be utilized for 1-click actions as well as downstream reporting, so it is important that these are correctly configured.

You may need to work with your developers to determine the best method to identify users. The most common methods to retrieve a User ID are:

* localStorage or sessionStorage item
* browser cookie
* accessing an item stored in the browser's dataLayer

Reach out to Redfast Support if you need assistance.

### FetchUserID (required)

This function is invoked to determine a user's authenticated User ID when the Redfast JS tag is loaded. An example implementation is below.

```javascript
static fetchUserId() {
  if( localStorage.getItem("ajs_user_id" )
    return JSON.parse(localStorage.getItem("ajs_user_id"));
  return null;
}
```

### FetchAnonymousUserID (optional)

This function is invoked to determine a user's anonymous User ID (if generated by your web app). If your web app does not generate an anonymous User ID, Redfast will automatically assign one to unauthenticated users.

```javascript
static fetchAnonUserId() {
  if( localStorage.getItem("ajs_anonymous_id" )
    return JSON.parse(localStorage.getItem("ajs_anonymous_id"));
  return null;
}
```
