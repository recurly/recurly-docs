Tags allow categorizing and tracking various prompts within the app, enabling better organization, and reporting of user interactions. 

You can create default tags at the app level to group prompts based on specific use cases, such as Engagement or Retention, or any other common criteria (e.g., market, offer, etc.). An app admin can add tags through Settings > Tags or on the prompt itself.

<Image align="center" className="border" border={true} src="https://files.readme.io/4a443a46f12b41134aed1a8ccdb932aba86cbf2b20f37945c75d01712dd475da-image.png" />

During prompt creation, you can select from the list of existing tags or add a new one under Details -> Tags. 

<Image align="center" className="border" border={true} src="https://files.readme.io/a248fe5e233d5c15ec140885651bc2a1ef2e0e33dceee6cbb084ce3cc6a5ae9c-image.png" />

Tags also allow to view combined performance metrics (e.g., impressions, conversions) for all prompts with a specific tag. Go to Prompts -> click on the filter icon next to the Tags column -> select the tag and view tag-specific stats along with the list of all prompts of this tag. 

<Image align="center" className="border" border={true} src="https://files.readme.io/ffa89a2acdc30406f18ce2a9655dac93d26eda3ca5e15a6e29ab57bf48213ba2-image.png" />

With tags, you can also control how often prompts with the same tag are shown to users in a set timeframe. To set the frequency cap by tag, go to Settings -> Tags -> click on the edit (pencil) icon for the tag in question -> select 'Custom' under Frequency Cap -> configure the Limit and hit 'Submit'.
