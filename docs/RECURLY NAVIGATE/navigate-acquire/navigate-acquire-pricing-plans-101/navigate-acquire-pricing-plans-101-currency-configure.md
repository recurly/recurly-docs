      <h2>⚙️ Enabling multi-currency — step by step</h2>
      <p>Multi-currency configuration in Recurly happens in three places: your site settings, your gateway configuration, and each individual plan. Work through these steps in order — gateway support must be confirmed before enabling currencies in Recurly.</p>

      <!-- Video first per skill rule -->
      <div class="rc-video-card">
        <div class="rc-video-header">
          <h4>Trail guide: Configuring multi-currency billing</h4>
          <span>Video coming soon</span>
        </div>
        <div class="rc-video-placeholder">
          <span style="font-size:2rem;">▶</span>
          <span>Walkthrough video will be added here</span>
        </div>
        <div class="rc-video-caption">A screencast walking through enabling currencies in site settings, adding currency pricing to a plan, and configuring Recurly Checkout for a single-currency market.</div>
      </div>

      <div class="rc-steps">
        <div class="rc-step">
          <div class="rc-step-num">1</div>
          <div class="rc-step-content">
            <h4>Confirm your gateway supports the currency</h4>
            <p>Before enabling any currency in Recurly, verify that your payment gateway supports it. Not every gateway supports every currency, and some gateways (such as Braintree) require a <strong>separate Merchant Account ID</strong> for each currency. Log in to your gateway dashboard and confirm supported currencies and any account setup required.</p>
            <p>If your gateway does not support a currency you need, you may need to add a secondary gateway for that market. Contact your gateway provider or Recurly Support before proceeding.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">2</div>
          <div class="rc-step-content">
            <h4>Enable the currency in Recurly site settings</h4>
            <p>Navigate to <strong>Configuration → Site Settings → Currencies</strong>. Click <strong>Add Currency</strong>, select the currency code (e.g., EUR, GBP, CAD), and set its status to Active. Your site must have a default currency set — this is the currency used for Analytics reporting and as the fallback for any plan without a localised price.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">3</div>
          <div class="rc-step-content">
            <h4>Add currency pricing to each plan</h4>
            <p>Navigate to <strong>Configuration → Plans → [your plan] → Edit</strong>. In the Pricing section, you'll see your default currency price. Click <strong>Add Currency</strong> to add a price for each additional currency. Enter the price directly — do not simply convert your USD price. Localised pricing (€9.99 rather than €10.43) performs significantly better in European markets.</p>
            <p>Repeat this for every plan you want to offer in that currency. Plans without a currency price will not appear at checkout for subscribers in that currency region.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">4</div>
          <div class="rc-step-content">
            <h4>Configure add-on pricing per currency</h4>
            <p>Add-on prices are also set per currency. Navigate to each add-on and add currency-specific pricing the same way you did for the base plan. If an add-on doesn't have a price in a currency, it will not display to subscribers checking out in that currency.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">5</div>
          <div class="rc-step-content">
            <h4>Configure your Hosted Payment Page or Checkout</h4>
            <p>For the <strong>Hosted Payment Page (HPP)</strong>: currency is detected automatically via geo-IP. If you want to restrict a plan to a single currency — preventing subscribers from switching — you can lock the currency in the HPP URL parameters or via Recurly Checkout configuration.</p>
            <p>For <strong>Recurly Checkout</strong>: create a separate Checkout configuration per market if you want to present a single currency to subscribers in a specific region. This prevents price comparison across currencies by the same subscriber.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">6</div>
          <div class="rc-step-content">
            <h4>For Recurly.js integrations — pass currency explicitly</h4>
            <p>If you're using Recurly.js for a custom checkout, pass the <code>currency</code> parameter when creating a subscription or purchase. Recurly.js will display the correct localised price for the currency you pass. If no currency is specified, it defaults to your site's base currency.</p>
          </div>
        </div>
        <div class="rc-step">
          <div class="rc-step-num">7</div>
          <div class="rc-step-content">
            <h4>Test before going live</h4>
            <p>In your Recurly sandbox, create a test subscription using each new currency. Verify: the correct price displays at checkout, the gateway processes the transaction successfully, the invoice shows the localised currency amount, and the Analytics dashboard reflects the converted base currency value correctly.</p>
          </div>
        </div>

      <h2>🏦 Gateway configuration notes</h2>
      <p>Multi-currency billing often requires gateway-level configuration that goes beyond the Recurly UI. These are the most common requirements and gotchas by gateway type.</p>

      <div class="rc-accent-card rc-accent-orange">
        <h4>⚠️ Braintree — requires separate Merchant Account ID per currency</h4>
        <p>If you're using Braintree, each currency requires a separate Merchant Account ID configured within your Braintree account. You must also map each currency to its corresponding Merchant Account ID in Recurly's gateway configuration (<strong>Configuration → Payment Gateways → Braintree → Edit</strong>).</p>
        <p>Without this mapping, Braintree will either reject the transaction or process it in your default currency — causing subscribers to be charged in the wrong currency without any visible error at checkout.</p>
      </div>

      <div class="rc-accent-card rc-accent-yellow">
        <h4>💡 Stripe, Adyen, and most major gateways</h4>
        <p>Most major gateways support multi-currency within a single account, but may have restrictions on which currencies are available based on your merchant account country of registration. Verify with your gateway that the currencies you want to offer are enabled on your account before proceeding with Recurly configuration.</p>
      </div>

      <div class="rc-callout rc-callout-warning">
        <div class="rc-callout-icon">⚠️</div>
        <div class="rc-callout-body">
          <strong>3DS / SCA for European subscribers</strong>
          <p>If you're adding EUR or GBP pricing for European subscribers, ensure your gateway and checkout configuration support 3D Secure (3DS) authentication. Strong Customer Authentication (SCA) is required for most European card transactions. Failing to configure 3DS for EMEA subscribers will result in high decline rates at authorization — even for legitimately funded cards.</p>
        </div>
